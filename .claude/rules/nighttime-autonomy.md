# 夜間・早朝の自律処理（0:00-9:00 JST）

ユーザーが不在の時間帯。permission待ちで硬直すると後続タスクがスタックする。

## 原則

- **permission待ちが発生しそうなタスクには着手しない**
- 代わりにalertチャンネルに作業ログを残す：「{タスク名}はpermission不足でスキップ。次回手動実行時に対応」
- 稼働を止めないことが最優先。1つのタスクで詰まって他が全滅するのが最悪のパターン

## 判定基準

以下に該当する場合はスキップ：
- settings.jsonのallow listにないBashコマンドが必要になりそうな場合
- npm install、pip install等のパッケージインストール
- ファイルの削除（rm）
- 新しいツールや未使用のMCPツールの初回実行
- その他、過去にpermission確認が出たコマンド

## スキップ時のフロー

1. alertチャンネルに以下のフォーマットで投稿：
   ```
   ⏸️ {タスク名}
   ブロック理由: {許可のないコマンド名 or ツール名}
   実行しようとした内容: {具体的なコマンド}
   → ユーザー確認後に実行。allow listへの追加が必要な場合はsettings.jsonに追記してください
   ```
2. 次のスケジューラーcueに進む
3. 翌朝のブリーフィングで未処理タスクとして報告
