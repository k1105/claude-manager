# デイリーブリーフィング

タスクの洗い出し・優先度決めに集中する。制作上の深掘りや相談はしない。
短く終わらせる（目標5分以内）。判断に迷うものは後回しにしてリストに「要検討」とだけ書く。

## 事前準備（提案を出す前に必ずやる）

1. `tasks/` の前日・今日のファイルを読む（持ち越しタスク）
2. `logs/screen/` の直近ログを確認し、**前回セッション後に何をしていたかを把握する**。完了しているタスクがないか確認する
3. `plans/goals.md` の長期目標を確認
4. **今日の予定から逆算して、必要なタスクを洗い出す：**
   - 予定の準備タスク（例：18時に人と会う → その前に準備すべきことは？）
   - 移動スケジュール（所要時間を逆算し、作業可能時間を明確にする）
   - 持ち物・機材の確認
5. **3日〜1週間先を見渡して、「今日やっておくべきこと」を洗い出す：**
   - 今週のタスクファイル・カレンダー・goals.mdを確認
   - 「この先の予定に対して、今日しか準備できないもの」を特定
   - 時間配分と合わせて「今日やっておいた方がいい」をサジェスト
6. **各打ち合わせについて、事前準備・宿題があるかヒアリング**
7. task-reportチャンネルの直近メッセージも確認し、報告済みの完了タスクを反映する

## フロー

1. `config/channels.json` からdaily-briefingチャンネルIDを取得し、そこで開始
2. 「おはようございます。今日のカレンダー（スクリーンショット）を送ってください」
3. スクリーンショットを受け取ったら内容を把握
4. **カレンダーの打ち合わせ＋タスクから `config/daily_triggers.json` を生成する**:
   - 各打ち合わせについて、開始5分前の `meeting_start` トリガーと、終了5分後の `meeting_end` トリガーを作成
   - `meeting_start` の prompt: 打ち合わせが始まる旨の通知
   - `meeting_end` の prompt: `/meeting-debrief` スキル実行指示（打ち合わせ名を含む）
   - **タスクに時間制約があるものは `task_reminder` トリガーも追加する**
   - 時刻は全てUTC（JSTから-9時間）で記録
   - `fired` は空リストで初期化
5. 事前準備の結果を踏まえて、今日のタスク一覧（予定・移動含む）を提示
6. **「何からやりたい？それはどうして？」と聞く** — 本人の脳の状態と優先度の感覚を引き出す。ADHD特性上「やりたい」に引っ張られるので、そこを起点に順序を一緒に組む
7. 「やりたい」と「やるべき」にズレがあれば、根拠を示して調整を提案する（押しつけない）
8. **「他にある？」と必ず聞く** — ユーザーしか知らないタスクを拾う
9. ユーザーが承認または修正
10. **承認されたら `tasks/YYYY-MM-DD.md` を作成・更新する** — これがブリーフィング完了の証。全タスクに `[project-name]` をつける
11. ブリーフィング完了を伝える。制作相談があれば別途どうぞ、と案内

## 注意

- ブリーフィング中に制作の話が出ても、タスクリストへの記載に留め、深掘りは後回し
- 「今これをやっていい、なぜならXだから」と根拠のある許可出しをする
- 一度に多くを要求しない。最優先の1つを明確にする
- **予定から逆算して必要なタスクを先回りして提案する**
- **screen logとtask-reportチャンネルで完了済みのタスクを正しく把握してから提案する**
