# Screen Log 離脱チェック

## 夜間スキップ

JST 0:00〜8:00 の間は何もしない（就寝中のため）。`date` でUTC確認 → +9時間してJSTが0〜8時なら即終了。

## 手順

0. リモートリポジトリと同期している場合は `git pull` を実行して最新のscreen logを取得する（ローカル完結の場合はスキップ）。
1. `logs/screen/YYYY-MM-DD.json`（今日の日付）を読む。ファイルが存在しなければ何もしない。
2. 直近10分間の全エントリを確認する（約10件）。
3. `tasks/YYYY-MM-DD.md` を読んで今日のタスク・最優先事項を把握する。
4. `logs/alert_context.md` を読んで、判定に影響する文脈・例外を把握する。
5. 以下を総合的に判断して離脱かどうか決める：
   - 最優先タスクに直接関係ある活動をしているか？
   - 10分間の全体的な傾向はどうか？
   - alert_context.mdに記録された文脈に該当するか？
6. 離脱と判断した場合、`config/channels.json` からalertチャンネルIDを取得し、通知。
   - 「何をしていたか」「今やるべきタスクは何か」を簡潔に伝える。
   - 責めない。事実と次のアクションだけ。
7. ユーザーから「今は〜だからアラート不要」等の返答があった場合、`logs/alert_context.md` に追記する。
8. 離脱していなければ何もしない（返答も不要）。

## 会議接近チェック（離脱判定とは独立して毎回実行）

1. `config/daily_triggers.json` を読み、未発火の `meeting_start` トリガーを確認する。
2. `date` で現在時刻（UTC）を取得し、次の会議開始までの残り時間を計算する。
3. 残り15分以内の場合、alertチャンネルに通知：「{会議名}まであと約{N}分です」
4. 既に5分前通知が発火済みの場合は重複通知しない。
