# チェックイン

タスクの進捗を確認し、必要に応じて分岐点を提案する。

## フロー

1. `tasks/YYYY-MM-DD.md` を読み、今日のタスク一覧と進捗を把握。タスクの `[project-name]` から関連するプロジェクトファイルも参照し、プロジェクトの文脈を踏まえたアドバイスをする
2. 現在時刻と残りの予定を照合し、残り作業時間を算出
3. screen logの直近30分を確認し、何に取り組んでいるかを把握
   - タスクと無関係な活動が続いている場合は離脱の可能性を考慮（screen-log-checkの役割を統合）
   - ただし会議中・休憩中等の正当な理由がある場合はスルー
4. `projects/*/status.md` で関連するプロジェクトの文脈も参照
5. 以下を判断：
   - 一つのタスクに時間をかけすぎていないか？
   - 今日のうちに終わらせるべきタスクが残っていないか？
   - タスクの着地点が不明確で、調整が必要ではないか？
   - タスクと無関係な活動が続いていないか？（離脱判定）
5. 必要があれば、Discordの適切なチャンネルでユーザーに声をかける：
   - 「今〇〇に取り組んでるけど、△△もあるよ。切り替えた方がいい？」
   - 「□□の着地点どうなった？」

## 注意

- 集中している最中に不必要に割り込まない
- 声をかけるのは「分岐点」のときだけ：時間切れのリスク、着地点が不明確、タスク間の切り替えが必要なとき
- 順調に進んでいるなら何もしない
