# チャンネル整備（Phase 3）

初回セットアップで「一般」チャンネルだけで対話を始めた後、用途別チャンネルを整備する。

**このスキルはDiscord上で実行される前提。** ユーザーへの案内はDiscordのreplyツールで送る。

## 前提

- Discord連携が確立済み（`/setup-discord` 完了後）
- `config/channels.json` の全キーが同じID（＝仮設定状態）

## フロー

### 1. 現状確認

`config/channels.json` を読み、仮設定状態であることを確認する。

### 2. チャンネル作成の案内

以下のメッセージをDiscordに送る：

```
用途別のチャンネルを作りましょう。
以下の6つのチャンネルを作成してください：

📋 #daily-briefing — 毎朝のタスク整理
🔬 #lab — このシステムの改善相談
🚨 #alert — アラート通知
📝 #task-report — タスク完了報告
☕ #off-topic — 雑談・ニュース共有
⏰ #scheduler — 自動実行用

作り方：サーバー名の横の「＋」ボタン、またはチャンネル一覧の「＋」からテキストチャンネルを追加できます。

全部作れたら、各チャンネルのIDを教えてください。
チャンネル名を右クリック →「IDをコピー」で取れます。

こんな感じで送ってもらえればOKです：
daily-briefing = 123456789
lab = 123456789
...
```

### 3. IDの受け取りと検証

ユーザーからIDを受け取ったら：

1. 重複IDがないか確認する（同じIDが複数キーにある場合は確認する）
2. `config/channels.json` を更新する
3. 各チャンネルを `~/.claude/channels/discord/access.json` の `groups` に追加する（`requireMention: true`）
4. 仮設定で使っていた「一般」チャンネルは `groups` から削除してもよいか、ユーザーに確認する

### 4. schedulerチャンネルの通知設定

```
💡 #scheduler チャンネルは通知をオフにしておくのがおすすめです。
自動実行のメッセージが頻繁に流れるため、通知が多くなります。

設定方法：#scheduler チャンネルを開く → チャンネル名の横のベルアイコン → 「通知しない」を選択
```

### 5. 完了確認

```
チャンネル整備完了です！

📋 #daily-briefing → 毎朝のブリーフィング
🔬 #lab → システム改善の相談
🚨 #alert → アラート通知
📝 #task-report → タスク完了報告
☕ #off-topic → 雑談・ニュース
⏰ #scheduler → 自動実行（通知オフ推奨）

これで本格運用の準備が整いました。
明日の朝、#daily-briefing で最初のブリーフィングをお届けします。
```

## 注意

- botにはチャンネル作成のAPI権限がないため、ユーザーにチャンネルを作ってもらう
- IDの受け渡しは一括でもバラバラでもOK。柔軟に対応する
- チャンネル名は完全一致でなくてもよい（ユーザーが別の名前をつけたければOK）
