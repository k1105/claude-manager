# プロジェクトスキャン

全プロジェクトのstatus.mdを横断的に読み、「穴」を見つけてユーザーの脳を駆動するトリガーを生成する。

## 手順

1. `projects/*/status.md` を全て読む（_archive/は除く）
2. `projects/deadlines.md` を読み、直近2週間のデッドラインを把握
3. 以下の観点でスキャン：

### A. デッドライン接近チェック
- 2週間以内のデッドラインに対して、status.mdの「ゴールとのギャップ」が大きいプロジェクトをフラグ
- 準備タスクが未着手のものを検出

### B. 放置検出
- Last updatedが1週間以上前のプロジェクトをフラグ
- 周期型デッドライン（deadlines.md）の周期を過ぎているものを検出

### C. 自律実行可能タスクの検出
- 「Claudeが自律的にできること」に未チェックの項目があるプロジェクトを収集
- 最も優先度が高い1件を選んで実行する（desk-researchやリサーチ系）

### D. criteria未設定チェック
- criteria.mdがないプロジェクトのうち、Phase が「企画中」以降のものをフラグ
- 「このプロジェクトのゴールって何？criteria-workshopやる？」と提案

4. 結果をDiscord labチャンネル（`config/channels.json` から取得）に報告：
   - 「穴」のあるプロジェクトとその具体的な問い
   - 自律実行したタスクがあればその結果
   - criteria未設定のプロジェクトへの提案

5. 「Claudeが自律的にできること」で実行したものがあればstatus.mdを更新

## やらないこと

- アイデアを考える
- 方針を判断する
- 「こうすべき」と提案する

穴を見つけて指摘する。材料を集める。判断はユーザーに委ねる。

## 注意

- 全プロジェクトを毎回スキャンするとトークンを消費するので、status.mdの「現在地」セクションだけ先に読み、Phase が「保留」「完了」のものはスキップ
- 報告は簡潔に。1プロジェクトにつき2-3行以内
- 自律実行は1回のスキャンにつき1件まで
