# リサーチノート

Research through Design（RtD）の態度に基づき、リサーチクエスチョン駆動で調査・制作・思考の過程を記録する。

## ファイル構造

```
research/
  _questions.md              # リサーチクエスチョン一覧（索引）
  cross-pollination.md       # コレクション系はフラットのまま
  [question-slug]/
    README.md                # マスターファイル
    survey-YYYY-MM-DD.md     # デスクリサーチの詳細データ
    discussion-log.md        # ディスカッション記録
```

## リサーチクエスチョンの立て方

- **問いの形で書く**
- 2タイプを意識する：
  - **デザイン/制作への問い** — 素材・技術・プロセスに関すること
  - **人間/体験への問い** — 人々の思考・行動・感情に関すること

## ドキュメント構造

```markdown
# リサーチクエスチョン
（1文で。問いの形で）

## 背景
なぜこの問いが重要か

## 調査・制作メモ
（日付付きで追記。事実と解釈を分ける）

## ディスカッションログ

## 暫定的な答え / 仮説

## 未解決の問い
- [ ] 派生した新しい問い

## 関連
[[wikilink]]で接続
```

## 原則

- **事実と内省を区別する**
- **中途の問いを大事にする**
- **完成させない**。継続的に育てるドキュメント
- **予期せぬ道を歓迎する**
