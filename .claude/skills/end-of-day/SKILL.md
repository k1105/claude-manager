# エンドオブデイ

## 手順

1. `tasks/YYYY-MM-DD.md` を読み、完了・未完了を確認
2. `logs/screen/YYYY-MM-DD.json` を読み、今日の作業内容を集計
3. `logs/YYYY-MM-DD.md` に今日のサマリーを追記：
   - 完了したタスク
   - 未完了タスク（理由があれば）
   - 今日の気づき・決定事項
   - 明日への引き継ぎ事項
   - **観察メモ**（Discordの会話ログをメインソースに、タスク進行と照合して振り返る）：
     - **今日のエピソード（ABC形式で1-2個）**: 先行条件→行動→結果
     - **トランジション**: タスク間の隙間に何が起きてた？
     - **起きなかったこと**: 予定してたのにやらなかった
     - **例外・驚き**: いつもと違ったこと
     - **一行所見**: 今日の最も重要な、非自明な観察
   - 観察メモの原則：
     - **会話ベースで分析する**。screen logは補助
     - まず事実を記述し、解釈は後
     - 責めない。パターンの発見と理解が目的
4. **プロジェクトファイルの更新チェック** — 今日の作業で `projects/` 配下のファイルに反映すべき進捗があれば更新する
5. 未完了タスクがあれば、翌日のタスクファイルに持ち越し準備
6. Discordで簡潔にサマリーを共有
7. リモートリポジトリと同期している場合は **git push**（ローカル完結の場合はスキップ）

## 注意

- 責めない。事実だけ淡々と記録する
- 「できなかった」ではなく「明日やる」というフレーミングで
