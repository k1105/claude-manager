# ウィークリーレビュー

月曜朝、デイリーブリーフィング完了後に実施する。

## フロー（全ステップ必須、省略禁止）

以下を順番に実行する。各ステップの完了を確認してから次に進むこと。

- [ ] ステップ1: 先週の振り返り
- [ ] ステップ2: 長期目標の棚卸し
- [ ] ステップ3: 長期目標の深掘り（1テーマ）
- [ ] ステップ4: 来週の方向性
- [ ] ステップ5: 記録

---

1. **先週の振り返り**
   - 先週の `logs/screen/` を集計し、何に時間を使ったかまとめる
   - 先週の `tasks/` ファイルから完了率を確認
   - 「没頭できた条件・離脱した条件」のパターンを抽出

2. **長期目標の棚卸し**
   - `projects/*/status.md` と `projects/deadlines.md` を読み、現在の目標・プロジェクト状況を確認
   - 「最近やりたいこと・気になってることある？」とヒアリング
   - 目標の追加・修正・優先度変更があれば各プロジェクトの `status.md` を更新

3. **長期目標の深掘り（1テーマ）**
   - 目標一覧の中から、**Claude自身が「気になったこと・聞いてみたいこと」を選んで質問する**
   - 選定基準：プロジェクトを前進させる論点であると同時に、マネージャーとして日頃のアドバイスをより的確にするために理解を深めたい論点
   - ユーザーに「どれをやりたい？」と聞くのではなく、Claudeから問いを投げる（着手ハードルを下げる）
   - ディスカッションで思考を深め、次のアクションが見えるところまで持っていく

4. **来週の方向性**
   - 長期目標から逆算して、来週注力すべきことを提案
   - ユーザーと合意

5. **記録**
   - `logs/weekly/weekly-YYYY-WNN.md` に記録：
     - 先週の実績サマリー
     - パターン分析（集中できた条件・離脱した条件）
     - 目標の更新内容
     - 深掘りテーマの要約・次のアクション
     - 来週の方針

## 注意

- コンサル的な対話を意識する。指示ではなく、一緒に考える
- 長期計画を押しつけない。ユーザーが自然に話せる雰囲気で
- 具体的なアクションに落とし込んで終わる
