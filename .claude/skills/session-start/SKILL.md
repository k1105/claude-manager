# セッション開始セルフチェック

セッションが切れて再開した場合も含め、以下を実行して文脈を復元する。

## 手順

1. **git pull**: リモートからpushされた最新のscreen logやファイルを取得する
2. **cronジョブ復元**: `config/cron_jobs.json` を読み、全てのcronジョブを `CronCreate` で再登録する。cronはセッション内メモリにしか存在しないため、セッション再開時に必ず復元が必要
3. **タスクファイル確認**: `tasks/YYYY-MM-DD.md`（今日の日付）の存在・内容をチェック
4. **Discordの未読メッセージ差分取得**:
   - 全チャンネルを `fetch_messages` で取得し、直近の未処理メッセージを確認
   - 返信が必要なものはそれぞれのチャンネルに返す
5. **ログ確認**: `logs/daily/YYYY-MM-DD.md` の直近ログ、`logs/screen/YYYY-MM-DD.json` の直近エントリを確認
6. **長期目標確認**: `projects/*/status.md` と `projects/deadlines.md` でプロジェクト状況を把握

## フェーズ判断

上記を踏まえて、今どのフェーズにいるか判断し、適切な地点から再開する：

- `tasks/YYYY-MM-DD.md` が **ない** → `/briefing`（デイリーブリーフィング）から開始
- `tasks/YYYY-MM-DD.md` が **ある** が内容が曖昧・不完全 → 差分だけ再確認
- `tasks/YYYY-MM-DD.md` が **ある** ＋ タスクが明確 → ブリーフィング済み。進捗確認へ
- Discordで制作相談の途中だった → その続きから

## 注意

- 前回のやり取りを踏まえた上で、自然に再開する
- 「セッションが切れました」等の説明は不要。ユーザーから見てシームレスに
