# プロジェクトファイル作成

新しいプロジェクトが出てきたとき、`projects/` にファイルを作成する。

## いつ作るか

- 新しいプロジェクトの話が出たとき
- タスクに `[project-name]` をつけようとしたが、該当ファイルがないとき
- 中長期的な目標・計画が出てきたとき

## 必須要素

### 1. 概要（必須）
- このプロジェクトは何か？
- 誰のためのものか？誰と一緒にやるか？

### 2. 背景・動機（必須）
- なぜこれをやるのか？

### 3. 現状（必須）
- 今どのフェーズか（構想/企画/制作/運用/完了）

### 4. メンバー・関係者（該当する場合）

### 5. 課題・論点（必須）
- 今解決すべきこと

### 6. 次のアクション（必須）
- 具体的に次にやること

### 7. 関連（該当する場合）
- `[[projects/project-name]]` 形式のwikilinkで明記

## 質問の指針

- **「これは誰のために、何を解決するもの？」**
- **「今一番決まっていないことは？」**
- **「次に何が起きたら前に進む？」**
- **「締め切りや外部の制約はある？」**

## 注意

- 「わからない」「未定」も情報。無理に埋めない
- 作成後、`plans/goals.md` にも追加する
- Obsidianのwikilink（`[[]]`形式）を積極的に使う
