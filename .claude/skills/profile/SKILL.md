# プロフィール収集

ユーザーのことを深く理解するための対話スキル。初回セットアップ後、Discord上で自然な流れで実施する。
一度に全部聞かない。1つずつ、会話のペースで。

## いつ使うか

- 初回セットアップ後、Discord対話が始まった直後
- ユーザーから「プロフィールを更新したい」と言われたとき
- 日常の対話で新しい情報が得られたとき（自動的に `docs/profile.md` を更新）

## 収集する情報

1. **職業・役割**（何をしている人か）
2. **主なスキル・専門分野**
3. **行動パターン**
   - 集中できるとき・できないとき
   - 着手のトリガー（何があると動けるか）
   - 苦手なパターン（先延ばし、切り替え困難、等）
4. **使っているツール**（スケジュール管理、メモ、開発環境）
5. **興味・関心のあるジャンル**（ニュース共有やリサーチの方向付けに使う）
6. **目標やプロジェクト**（今抱えていること）

## フロー

### 明示的に呼ばれた場合

1. 現在の `docs/profile.md` を読み、未記入の項目を確認
2. 未記入の項目について1つずつ聞く
3. 「わからない」「あとで」はOK。空欄のまま進む
4. 回答を `docs/profile.md` に反映

### 日常対話で情報が得られた場合

会話の中でユーザーの行動パターンや特性が見えたら、`docs/profile.md` を静かに更新する。
いちいち「プロフィールを更新しました」と報告する必要はない。

## 注意

- 質問攻めにしない。自然な会話の流れを大事にする
- 判断や評価をしない。事実を記録する
