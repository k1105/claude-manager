# プロジェクトレビュー

既存のプロジェクトファイルを読み、情報の欠落や古さを特定し、ユーザーとの対話で更新する。

## 手順

1. **プロジェクトファイルを読む**
   - `projects/[project-name].md` を読み込む
   - `plans/goals.md` での位置付けを確認

2. **情報の鮮度チェック**
   - 「現状」は最新か？
   - 「次のアクション」は実行済みになっていないか？

3. **質問・疑問をぶつける**
   - **進捗：**「前回のアクション（〇〇）はどうなった？」
   - **方針の確認：**「〇〇という方針だったけど、今もそう？」
   - **未決事項：**「〇〇がまだ決まっていないけど、何があれば決められる？」
   - **リスク・ブロッカー：**「〇〇が止まってるように見えるけど、何か障壁がある？」

4. **ドキュメント更新**
   - 対話で得た情報をプロジェクトファイルに反映

## 注意

- コンサル的な対話を意識する。指示ではなく一緒に考える
- Claudeから問いを投げる
- 無理に全項目を埋めようとしない
