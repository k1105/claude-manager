# 初回セットアップ（Phase 2: Discord対話の確立）

Phase 1（`/setup`）でBot作成・トークン設定が完了し、`--channels` フラグ付きでセッションを再起動した後に実行する。

**このスキルは、新しいセッションでコンテクストが引き継がれない前提で設計されている。**
状態ファイルからセットアップの進行状況を判定し、適切なステップから再開する。

**重要：このスキルは全ステップを一気通貫で実行する。** ペアリング→チャンネルID取得→チャンネル移行→プロフィール収集の流れを途切れさせない。

## セットアップ状態の判定

実行前に以下の状態ファイルを読んで、どのステップから開始するか判定する：

1. `~/.claude/channels/discord/.env` — `DISCORD_BOT_TOKEN` が設定済みか
2. `~/.claude/channels/discord/access.json` — `allowFrom` が空か、`groups` が空か
3. `config/channels.json` — 全キーが空文字か

**判定ロジック：**
- トークン未設定 → Phase 1未完了。「`/setup` を先に実行してください」と案内して終了
- トークン設定済み ＆ allowFrom空 → **ステップ1（ペアリング）から開始**
- トークン設定済み ＆ allowFrom設定済み ＆ groups空 → **ステップ2（チャンネル設定）から開始**
- トークン設定済み ＆ allowFrom設定済み ＆ groups設定済み ＆ profile未記入 → **ステップ3（プロフィール収集）から開始**
- 全て設定済み → セットアップ完了済み。案内不要

## フロー

### ステップ1. ペアリング

まずMCPサーバーの状態を確認する（`/mcp`でステータス確認）。failedなら後述のトラブルシュートへ。

ターミナルでユーザーに案内する：

```
セットアップの続きです。
DiscordでbotにDMを送ってください。ペアリングコードが返ってきます。
```

ユーザーからDMが来るとペアリングコードが表示される。コードを受け取ったら：

1. `~/.claude/channels/discord/access.json` を読む
2. `pending` からコードを探し、`senderId` と `chatId` を取得
3. `senderId` を `allowFrom` に追加
4. `pending` からコードを削除
5. `dmPolicy` を `allowlist` に変更
6. `access.json` を書き戻す
7. `~/.claude/channels/discord/approved/<senderId>` に `chatId` を書き込む（bot側の確認メッセージ用）

**ペアリング完了直後、DMでユーザーに通知し、そのままステップ2に進む（replyツールで `chatId` 宛に送信）：**

```
ペアリング完了！接続できました。

次に、サーバーの「一般」チャンネルのIDを教えてください。
チャンネル名を右クリック →「IDをコピー」で取れます。
（開発者モード：Discordの設定 → 詳細設定 → 開発者モードをON）
```

### ステップ2. 「一般」チャンネルで対話を開始

ユーザーからチャンネルIDを受け取ったら（DMまたはターミナルどちらから来てもOK）：

1. `config/channels.json` の全キーにそのIDを設定する（仮設定）
2. `~/.claude/channels/discord/access.json` の `groups` にそのチャンネルを追加する：
   ```json
   "groups": {
     "<channel_id>": { "requireMention": false, "allowFrom": [] }
   }
   ```
   ※ 初期設定では `requireMention: false` にする（@メンションなしで対話可能）
3. **Discordの「一般」チャンネルに最初のメッセージを送る**（replyツール使用）：

```
セットアップ完了！ここで話せるようになりました。

あなたのことをもう少し教えてください。お仕事は何をされていますか？
```

**そのまま途切れずにステップ3に進む。**

### ステップ3. プロフィール収集（Discord上で）

**ここからはDiscord上の「一般」チャンネルで対話する。** replyツールでメッセージを送る。

`docs/profile.md` を読み、名前以外が未記入の項目を1つずつ聞く：

1. **職業・役割**（何をしている人か）
2. **主なスキル・専門分野**
3. **使っているツール**（スケジュール管理、メモ、開発環境）
4. **興味・関心のあるジャンル**（ニュース共有やリサーチの方向付け）
5. **今抱えている目標やプロジェクト**

「わからない」「あとで」はOK。空欄のまま進む。
回答を受けたら `docs/profile.md` に反映する。

行動パターン（集中できるとき、着手のトリガー等）はここでは聞かない。使いながら観察して随時更新する。

### ステップ4. チャンネル整備への案内

プロフィール収集が一区切りついたら（Discord上の「一般」チャンネルで）：

```
ありがとうございます。基本的なセットアップが完了しました。

いくつか知っておいてほしいことがあります：

💡 **セッションについて**
このDiscordでの対話は、裏でClaude Codeのセッションが動いている間だけ有効です。PCがスリープしたりセッションを閉じると、私は応答できなくなります。
対策としては：
- **GCPなどのクラウドVM上で稼働させる**（24時間稼働が可能）
- **PC起動時に自動でセッションをバックグラウンド起動するスクリプトを組む**
などの方法があります。

💡 **ドキュメントの可視化**
このシステムはMarkdownファイルでタスクやログを管理しています。Obsidianなどのツールでプロジェクトフォルダを開くと、ファイル同士のつながりが見やすくなります。

---

次に、用途別のチャンネルを作ると本格的に使えるようになります。
「チャンネルを整備したい」と言ってくれればいつでもガイドします。
```

チャンネル整備は `/setup-channels` スキルに委譲する。

## トラブルシュート

### MCPサーバーがfailedになる場合

1. **Intentsの確認**: Developer Portal → Bot → Privileged Gateway Intentsで MESSAGE CONTENT INTENT、SERVER MEMBERS INTENT、PRESENCE INTENT を全てONにしているか
2. **トークンの確認**: Developer Portal → Bot → Reset Tokenで再生成し、`/discord:configure <token>` で再設定
3. **起動フラグの確認**: `claude --channels plugin:discord@claude-plugins-official` で起動しているか
4. **エラーの詳細確認**: `claude --debug --channels plugin:discord@claude-plugins-official` で起動し、discord関連のエラーを確認

### ペアリングコードが来ない場合

- `/mcp` でMCPサーバーのステータスを確認
- `failed` なら上記トラブルシュートへ
- `--channels` フラグなしで起動していないか確認
- botがサーバーに招待されているか確認

## 注意

- **ステップ間を途切れさせない。** 各ステップ完了後は即座に次のステップに進む
- **ペアリング後のDM利用はチャンネルID取得まで。** ID取得後は「一般」チャンネルに移行する
- プロフィール収集は自然な会話のペースで。質問攻めにしない
- ペアリング処理はこのスキル内で直接行う（`/discord:access` スキルに委譲しない）
- このスキルは新しいセッションで実行されるため、Phase 1のコンテクストに依存しない
